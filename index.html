.יוספה כהן (סבתי מצד אמא) נולדה בתאריך ה-30.6.1939 במושבה מצפה שליד טבריה
.יוספה הייתה הבת הרביעית להוריה, שלמה ויפה, ויש לה 3 אחים. הוריה התכוונו לקרוא לה שלומית אולם לפני שנולדה בקשה אחת מקרובות המשפחה לקרוא לילד שיוולד על שם בעלה יוסף שנפטר. וכשנולדה קראו לה יוספה
הוריה הגיעו מפולין בשנות ה- 20. לאביה היה קצת כסף והוא התלבט האם לקנות מגרש בתל אביב או 2 פרות. אחיו שהגיע לפניו לארץ היה חקלאי ולכן הוא החליט ללכת בעקבותיו ולהפוך לאיכר. הם קנו 2 פרות ואדמה והקימו את ביתם במושבה מצפה שהייתה אחת ממושבות הברון רוטשילד. המושבה הייתה קטנה מאוד והיו בה רק 7 משפחות. סבתא הייתה הילדה היחידה בכל המושבה, שאר הילדים היו בנים. סבתא לא בקרה בגן ילדים כי לא היה כזה במושבה .מכיוון שהיה מספר מועט של ילדים, היה רק מורה אחד שלימד ילדים מ-3 כיתות שונות. כל הילדים למדו בחדר אחד, כל קבוצת גיל בפינה אחרת של החדר. סבתא הייתה התלמידה היחידה בכתה א'. כשהגיע לכתה ב' סגרו את בית הספר במושבה והיא הייתה צריכה ללכת לבית ספר בכפר חיטים שהיה מרוחק כ- 2 ק"מ מביתה.
.בבית בשנים הראשונות לא היה מים זורמים ולא חשמל. מים היו מביאים בחביות בעזרת עגלה רתומה לסוס ממושב כפר זיתים והיו משתמשים במנורות נפט לחימום ותאורה
.ליוספה לא היו צעצועים למשחק וכשהיא הצטלמה (צילום יחיד כילדה) נתנו לה להחזיק בובה של הצלם
כל אנשי המושבה עבדו בחקלאות ולכל משפחה הייתה רפת עם כמה פרות ולול אפרוחים. הילדים הם שבחרו את השמות לפרות סבתא זוכרת שהשמות היו: עטרה, עידית, גלילה וחמדה
בגיל 16 עברה יוספה ללמוד בקיבוץ תל יוסף והייתה ילדת חוץ למשך שנתיים. בצבא שרתה בחיל האוויר ולאחר מכן .החלה ללמוד בבית ספר לאחיות בתל השומר, שם פגשה בחולה שעבר תאונת אופנוע ושמו סמי. ולאחר מספר שנים, בסיום לימודייה הם התחתנו
.יוספה עבדה במשך כ-40 שנה כאחות בבית החולים בלינסון במחלקת הדיאליזה, ולפני כ-16 שנים יצאה לפנסיה בהמשך חייה נולדו ליוספה שלושה ילדים, סיגל (אימי), מירב ויאיר. יאיר נהרג בפעולה צבאית ברצועת עזה בשנת 1994, דבר אשר ערער את חייה. במשך השנים נולדו ליוספה חמישה נכדים, אמיר, אור (אני), שחר, זוהר וזיו. בשנת 2003, נפטר בעלה, סמי, וכיום היא עדיין מתגוררת בביתה ונעזרת במטפלת סיעודית. בשנים האחרונות, ובמיוחד בתקופת הקורונה אנחנו הנכדים באים לבקרה להיות איתה, ולשמחה כדי שלא תרגיש בודדה.

the web is Read-Only and static.

the web is wildly Read-Write.

the web is portable and personal - Read-Write intelligent web.

search engines retrieve macro contents. Search is very fast but many times results are inaccurate or more than users can chew.

search engines retrieve tags with micro contents. The process of tagging is manual, tedious and covers negligible percents of the WWW.

search engines will retrieve micro content texts which were tagged automatically.This implies translating billions of Web 1.0 macro contents into micro contents. The result could be more precise search because tagging can solve part of the ambiguity that homonyms and synonyms introduce into the process of Search.

Companies publish content that people consume (e.g. Ynet)

People publish content that other people can consume, companies build platforms that let people publish content for other people (e.g. YouTube, Wikipedia)

People build applications that people can interact with, companies build platforms that let people publish services by leveraging the associations between people or special content ( e. g. Facebook, Instagram)



Use Case example - noa kirel
  • Web 1.0 - for finding the relevant data we want about "noa kirel - latest song", the user have to write a specific URL.
  • Web 2.0 - for finding result we can use "noa kirel - latest song" as a query to a search engines and we will get result from the query as web tags.
  • Web 3.0 - for finding result we can use "noa kirel - latest song" as a query to a search engines and we will get result that contain the query words and semantic words as tags or web info.
